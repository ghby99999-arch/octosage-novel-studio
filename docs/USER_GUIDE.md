# User Guide

## Workspace

OctoSage keeps a local workspace path for the current machine. You can change it in `设 · 系统配置`.

## Login State

The bottom-left account area stores the local account display state. It is meant to make the desktop shell feel persistent without forcing a cloud sign-in before writing.

## Writing Flow

The normal writing loop is:

1. Create or open a project.
2. Write one chapter.
3. Review it.
4. Rewrite if needed.
5. Export merged text or keep going to the next chapter.

The app also supports batch writing so the user can let it continue for several chapters at once.

## Quality Flow

Use the quality report and premium gate when you want a chapter to pass the current internal quality standard before export or publish.

## Video Flow

The video area exports:

- screenplay
- storyboard
- prompts
- character reference prompts
- scene reference prompts
- full video package

This is designed for AI video tools that need stable characters and clear shots.

## Publish Flow

The publish area creates a package and a browser-assisted publish helper.

Important:

- The final submit step stays under user control.
- The app does not silently execute remote install scripts.
- The browser helper is meant to help with visible, logged-in sessions only.

## Support

The settings page includes:

- quick start
- user guide
- changelog
- diagnostic export
- workspace controls
- support WeChat copy button

When reporting a problem, export the diagnostic file and send that together with the project description.
